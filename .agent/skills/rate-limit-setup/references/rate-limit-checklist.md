# Rate Limit Checklist

## 1. 目标接口

优先考虑：
- 登录 / 注册 / 验证码
- 密码重置
- Webhook / callback
- 公开查询接口
- 高成本 AI / 报表 / 导出接口

## 2. 技术接入点

- 是否存在全局 limiter 配置
- 是否在 router 上声明限额
- 是否有统一 429 响应
- 是否处理代理 / client IP 获取问题

## 3. 豁免与特例

- `/health`
- `/metrics`
- 内部管理接口（按具体安全边界决定）

## 4. 测试

- 至少一条触发 429 的测试
- 至少一条未触发限制的正常请求测试
- 文档说明限流策略的适用范围
